# Hybrid Rules

1. Consent is required before changing source projects.
2. SkyCorePi and BloomStudio remain intact.
3. The hybrid repo is the test lab.
4. Each imported idea gets a note explaining why it was chosen.
5. One system gets tested at a time.
6. Broken parts go to legacy notes, not the trash.
7. Restore points are written before major changes.
8. No agent identity, project marker, or symbolic name gets flattened.

## Adapter Rule

When two systems disagree, create an adapter instead of forcing one project to become the other.
